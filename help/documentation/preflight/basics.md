# プリフライトの基本

![プリフライト](./assets/overview/hero.png){align="center"}

プリフライトは、web ページを公開する前に、web ページを改善する機会を特定するのに役立ちます。 Preflight 拡張機能は、コンテンツに対して監査を実行することで機会を識別し、その結果をパネルに表示するので、公開前に対処できます。

## プリフライトが表示される場所

プリフライトは、様々なオーサリング環境で使用できます。

* **ユニバーサルエディター** - プリフライト拡張機能が **サイドパネル** に表示されます。 現在のページの監査を開始するには、これを選択します。
* **ドキュメントベースのオーサリング** - Sidekick（ブックマークレット）を使用して、プレビューしたページコンテンツに対してプリフライトツールを実行し、オポチュニティのリストを確認します。
* **AEM Sites ページエディター** - ブラウザーで Preflight ブックマークレットを使用して、監査を開始します。

設定手順については、「プリフライト設定 [&#x200B; を参照してください &#x200B;](./setup.md)。

## 監査の開始

プリフライトを実行するには：

1. 監査するページをオーサリング環境（ユニバーサルエディター、ドキュメントベースのプレビュー、AEM Sites ページエディター）で開きます。
2. プリフライトパネルを開く – サイドパネルからプリフライト拡張を選択するか、Sidekickで「プリフライト」ボタンをクリックします。
3. プリフライトはページを分析し、見つかった機会を表示してページを改善します。

## 監査結果

監査が完了すると、検出されたオポチュニティがプリフライトに表示されます。 各オポチュニティはタイプ別に整理されており、問題の解決方法の詳細が含まれています。

AEMのプリフライトダイアログの上部にあるは、全体的な監査結果を反映したユーザーの進行状況バーです。 問題なく成功した商談の割合と、すべての商談で見つかった問題の合計数が表示されます。 ユーザーの進行状況バーは、作成者がページの全体的な正常性を一目で把握するのに役立ちます。

## プリフライトの機会について

Preflight では、アクセシビリティ、メタデータ、リンク、読みやすさなど、コンテンツの複数の側面を評価します。 使用可能な商談タイプの完全なリストとその対処方法については、[&#x200B; プリフライトの商談 &#x200B;](./overview.md) を参照してください。
